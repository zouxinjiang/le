# 需求分析
## 需求
1. 类似于记事本之类的东西，能够随时记录发生的事情
2. 能够树立flag，然后将事件与flag相互关联
3. 能够上传图片，视频
4. 能够随时对事件进行梳理，总结

## 模块
1. 登录模块
    - 能够自我注册账户
    - 能够使用微信登录
    - 验证码
2. 权限模块(使用url进行权限的区分)
    - 普通用户
    - vip用户
    - 超级用户
3. 用户模块
    - 普通用户
    - 超级会员
    - CRUD
4. 缓存模块
    - 内存缓存
    - 文件缓存
    - 数据库缓存
5. 文件管理模块
    - 相同文件不能存多份
6. 业务模块
    - 事件CRUD
    - flag CRUD
